# Skill: Deliver

## Purpose

This skill defines how to deliver the completed digest to the user via email (Gmail MCP) and optionally WhatsApp (Twilio).

---

## Step 1: Read Configuration

Read `context/delivery_config.md` for:
- Recipient email address
- Subject line format
- WhatsApp configuration (if present)

---

## Step 2: Send Email via Gmail MCP

### Prerequisites
- The Gmail MCP server must be authenticated in Claude Code
- If not authenticated, skip email delivery and print instructions for setup

### Process
1. Read the HTML content generated by the `digest_writer` skill
2. Use the Gmail MCP `send_email` tool (or equivalent) with:
   - **To:** Recipient email from delivery config
   - **Subject:** "AI Research Digest — YYYY-MM-DD"
   - **Body:** HTML content from the digest writer
3. Confirm the email was sent successfully

### If Gmail MCP is Not Available
If the Gmail MCP tool is not available or not authenticated:
1. Print a clear message: "Gmail MCP not authenticated. Digest saved to output/YYYY-MM-DD-digest.md"
2. Print setup instructions:
   - "To enable email delivery, authenticate the Gmail MCP server in Claude Code settings"
   - "The digest has been saved locally and can be read directly"
3. Do NOT fail the full run — the digest is still saved as a file

---

## Step 3: Send WhatsApp Summary (Optional, Phase 2)

Only attempt if WhatsApp credentials are configured in `context/delivery_config.md`.

### Process
1. Compose a short text-only summary from the highlights:

```
AI Research Digest — YYYY-MM-DD

Top 3 Highlights:
1. [Title] — [one-line why it matters]
2. [Title] — [one-line why it matters]
3. [Title] — [one-line why it matters]

Podcast pick: [Episode name] by [Podcast]

Full digest in your email.
```

2. Send via Twilio API using curl:

```bash
curl -X POST "https://api.twilio.com/2010-04-01/Accounts/${ACCOUNT_SID}/Messages.json" \
  --data-urlencode "Body=${MESSAGE}" \
  --data-urlencode "From=whatsapp:+14155238886" \
  --data-urlencode "To=whatsapp:${TO_NUMBER}" \
  -u "${ACCOUNT_SID}:${AUTH_TOKEN}"
```

3. If WhatsApp is not configured, skip silently (this is optional)

---

## Step 4: Confirm Delivery

Print a summary of what was delivered:

```
Delivery complete:
- Markdown digest: output/YYYY-MM-DD-digest.md ✓
- Email: [sent/skipped] to [recipient]
- WhatsApp: [sent/skipped/not configured]
```

---

## Error Handling

- If email fails: save the digest file, report the error, suggest checking Gmail MCP auth
- If WhatsApp fails: report the error but do NOT retry — it's optional
- Never fail the entire run because of delivery issues — the digest file is always the primary output
