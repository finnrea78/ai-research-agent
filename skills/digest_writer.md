# Skill: Digest Writer

## Purpose

This skill defines how to compose a daily AI research digest from filtered and ranked findings. It produces both a Markdown file (for archival) and HTML content (for email delivery).

---

## Step 1: Compose Markdown Digest

Write the digest to `output/YYYY-MM-DD-digest.md` using this structure:

```markdown
# AI Research Digest — YYYY-MM-DD

> Scanned N sources. N items passed the signal filter (threshold: N/25).

---

## Highlights

### 1. [Item Title](URL)
**Source:** [source name] | **Score:** N/25 | **Published:** YYYY-MM-DD

[Why this matters — 1-2 sentences]

**Key takeaway:** [Single most important point]

**Action:** [What to do — listen, read, try, star]

---

[Repeat for remaining 2-4 highlights]

---

## Podcasts

| Episode | Podcast | Duration | Published |
|---------|---------|----------|-----------|
| [Title](URL) | Podcast Name | 45 min | YYYY-MM-DD |

**Recommended:** [Episode name] — [brief reason why]

---

## Repos & Tools

| Repo | Stars | Language | What it does |
|------|-------|----------|-------------|
| [owner/name](URL) | 1.2k | Python | One-line description |

---

## News & Analysis

- **[Article Title](URL)** — [1 sentence summary]. *[Author/Source]*
- **[Article Title](URL)** — [1 sentence summary]. *[Author/Source]*

---

## Discovered Sources

These sources were found during this scan and are not in the seed list:

- **[Source Name](URL)** — [What it is and why it showed up]

---

*Generated by AI Research Agent on YYYY-MM-DD*
```

---

## Step 2: Writing Principles

Follow these rules for all digest text:

1. **Lead with what's interesting** — don't bury the hook
2. **Be specific** — "Claude Code 1.5 adds native MCP debugging" not "Claude Code was updated"
3. **Include numbers** — star counts, episode lengths, point counts
4. **State the "so what"** — why should the reader care?
5. **Keep it scannable** — tables, bold text, short sentences
6. **No hype language** — no "game-changing", "revolutionary", "mind-blowing"
7. **Max 2 sentences per item** (except highlights which get 3-4)

---

## Step 3: Generate HTML Version

Convert the Markdown digest into HTML using the template at `templates/email_template.html`.

Replace template placeholders:
- `{{DATE}}` → current date
- `{{HIGHLIGHTS}}` → rendered highlights section
- `{{PODCASTS}}` → rendered podcasts table
- `{{REPOS}}` → rendered repos table
- `{{NEWS}}` → rendered news list
- `{{DISCOVERED}}` → rendered discovered sources
- `{{STATS}}` → scan metadata (sources scanned, items passed, threshold)

The HTML version is passed to the `deliver` skill.

---

## Step 4: Save and Confirm

1. Save the Markdown digest to `output/YYYY-MM-DD-digest.md`
2. Confirm the file was written successfully
3. Pass both the Markdown content and HTML content to the `deliver` skill
